# COMP 690 — Steered CLIP via VLM-Guided Patch Selection

This folder contains the code to reproduce the two main experiments from the
paper:

1. **Fine-grained classification on CUB-200-2011** with per-class part queries.
2. **Yes/No VQA on VQAv2** under the TAP-C scoring protocol.

Both experiments share the same PASTA-style attention steering implementation
in `ConformationCLIP_ViT.py`.

---

## 1. Prerequisites

### Python packages

```
pip install -r requirements.txt
```

### External assets

Point the scripts at your data via two environment variables. (If unset, they
fall back to `./CUB_200_2011/` and `~/data/vqa/`.)

```
export CUB_DATA_ROOT=/path/to/CUB_200_2011
export VQA_DATA_ROOT=/path/to/vqa
```

**CUB-200-2011** layout (under `$CUB_DATA_ROOT`):
```
images/<class_id>.<class_name>/*.jpg
image_class_labels.txt
images.txt
```

**VQAv2 val** layout (under `$VQA_DATA_ROOT`):
```
v2_OpenEnded_mscoco_val2014_questions.json
v2_mscoco_val2014_annotations.json
images/val2014/COCO_val2014_*.jpg
```

**Grounding DINO weights** are downloaded automatically by `load_grounding_dino`
on first run.

### Ollama (for VLM-generated queries)

```
ollama pull gemma4:e4b      # used for VQA DINO queries + TAP-C templates
ollama pull qwen2.5vl:7b    # used for CUB per-class part queries
ollama serve                # leave running in a separate terminal
```

## 3. Reproducing the CUB result

```
python precompute_birds_cache_perclass.py    # ~30 min on CPU; one-time
python test_birds_perclass.py                # ~2 min once cache exists
```

Settings (in `test_birds_perclass.py`):

- `CACHE_SAMPLES = 3000`, `N_PROFILE = 500`, `N_EVAL = 2500`
- `PASTA_ALPHAS = [0.3, 0.1, 0.05]`, `HEAD_COUNTS = [5, 10]`
- `PROFILE_LAYERS = list(range(4, 12))`, `PROFILE_ALPHA = 0.1`

Expected output (top-10 heads, α=0.3): plain CLIP **47.68%**, PASTA **48.00%**
(Δ = **+0.32%**).

---

## 4. Reproducing the VQA result

```
python precompute_tapc_cache.py              # ~1.5 hr on CPU; one-time
python test_tapc_pasta.py                    # ~20 min once cache exists
```

Settings (in `test_tapc_pasta.py`):

- `PRECOMPUTE_POOL_SIZE = 1500`, `N_PROFILE = 500`, `N_EVAL = 1000`
- `ALPHAS = [0.02, 0.05, 0.1]`, `HEAD_COUNTS = [5, 10, 20]`
- `PROFILE_LAYERS = list(range(6, 12))`, `PASTA_ALPHA = 0.1`

Expected sweep result: plain TAP-C **62.10%**, best PASTA (top-20, α=0.1)
**61.10%** (Δ = **−1.00%**).

---

## 5. Notes

- The CUB experiment uses the `perclass` cache tag (no patch cap, no DINO
  tightening) — this matches the +0.32% headline result.
- The VQA experiment uses precompute cache version 1; bump to 2 only if
  regenerating with the multimodal Gemma prompt in `dino_prompt.py`.
- All steering preserves the CLS token in the selected set and only steers
  the CLS *query row* of each chosen attention head (see
  `_steered_block_output` in `ConformationCLIP_ViT.py`).
